**状态：** 初稿

## 1. 系统架构概述

### 1.1 架构设计原则
- **分层架构**：表现层、业务层、数据访问层分离
- **模块化设计**：高内聚、低耦合
- **微服务准备**：模块间通过接口通信，便于后期拆分

### 1.2 技术栈建议
- **后端**：Spring Boot
- **前端**：Vue.js
- **数据库**：MySQL + Redis
- **消息队列**：RocketMQ

## 3. 模块详细设计

### 3.1 用户管理模块 (User Module)

#### 3.1.1 模块功能
- 用户注册、登录、注销
- 用户信息管理（基本信息、收货地址）
- 分销商资格管理
- 用户认证与授权

#### 3.1.2 核心接口

| 接口名称 | 方法 | 路径 | 描述 |
|---------|------|------|------|
| 用户注册 | POST | /api/user/register | 用户注册账号 |
| 用户登录 | POST | /api/user/login | 用户登录获取token |
| 获取用户信息 | GET | /api/user/profile | 获取当前用户信息 |
| 更新用户信息 | PUT | /api/user/profile | 更新用户基本信息 |
| 管理收货地址 | CRUD | /api/user/addresses | 收货地址增删改查 |
| 申请分销商 | POST | /api/user/distributor/apply | 申请成为分销商 |
| 分销商信息 | GET | /api/user/distributor/info | 获取分销商信息 |

#### 3.1.3 依赖关系
- **依赖**：系统配置模块（获取注册配置）
- **被依赖**：订单模块、分销模块、财务模块

## 6. 模块通信机制

### 6.1 同步调用（RESTful API）
- 模块间直接HTTP调用
- 适用场景：实时性要求高的操作（如创建订单时校验库存）

### 6.2 异步消息（Message Queue）
- 使用消息队列解耦
- 适用场景：
  - 订单支付成功后触发返利计算
  - 订单状态变更发送通知
  - 数据统计异步处理

### 6.3 事件驱动
```java
// 示例：订单确认收货事件
@EventListener
public void onOrderConfirmed(OrderConfirmedEvent event) {
    // 1. 触发返利计算
    distributionService.calculateCommission(event.getOrderId());
    
    // 2. 发送通知
    messageService.sendOrderConfirmNotification(event.getUserId());
    
    // 3. 更新统计
    reportService.updateSalesStats(event.getOrderId());
}
```

## 9. 附录

### 9.1 模块接口规范
- RESTful API 设计
- 统一响应格式：
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1697356800000
}
```

### 9.2 错误码规范
- 1000-1999：用户模块
- 2000-2999：商品模块
- 3000-3999：订单模块
- 4000-4999：分销模块
- 5000-5999：财务模块

### 9.3 安全规范
- 所有接口需JWT认证（公开接口除外）
- 敏感操作需二次验证
- 接口限流防刷
- SQL注入防护
